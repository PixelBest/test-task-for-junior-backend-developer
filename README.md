Решение задачи по собеседованию:

Первым делом добавил soft-delete для task, в базе должна храниться вся история записей

Убрал из запроса по созданию и изменению status, сделал отдельные запросы для изменения статуса, отдельно для in_progress и отдльено для done, подразумевается что в приложении рядом с task будет сначал кнопка "Взять в работу", а потом "Сделана", чтобы не заходить каждый раз в редактирование и изменять статус задачи, так же на скриншотах к заданию была колонка "Срок выполнения", в задании и в коде она не прописана, но она может понадобится в будущем для историчности и формирования например отчёта по сделанным задачам, с моей точки зрения лучше будет использовать отельную таблицу history для записи всех действий работников

Я добавил колонки:

periodicity_type - выбрать периодичность (ежедневная, ежемесячная и др);

periodicity - для значения по переодичностям. Если periodicity_type=daily, то в это поле записывается ежедневная переодичность, то есть например каждые 2 дня будет создаваться новая задача. Если Если periodicity_type=dailyЕсли periodicity_type=monthly, то записывается число месяца. Если periodicity_type=parity, то записывается 0 или 1, чётные и нечётные соответственно;

periodicity_dates - массив дат, конкретные даты, когда будут создаваться задачи;

periodicity_closed - булевая колонка по закрытию переодичности, если надобность в переодичности больше не нужна, можно либо в редактировании просто убрать переодиночсть, либо я сделал отдельный endpoint который по id записи убирает у неё переодичность, подразумевается как отедльная кнопка на вебе;

parent_id - идентификатор родителя, когда создаются новые задачи по переодиночсти, в неё записывается родитель, так можно отследить историчность.

Логика переодичности: я сделал backgraund процесс, который будет активироваться каждый день в 00:00, брать все данные по которым нужно сделать новые задачи по переодичности, проверяет по дате и типу переодичности нужно ли создавать новые задачи или нет, и создаёт при надобности. Старой задаче ставится periodicity_closed=true, и проверка на следующий день будет происходить по новой задаче. Если тип переодичности по конкретным датам и задача сделана по максимальной дате, то у новой задачи тоже ставится periodicity_closed=true.
